# UI Mobile Web Implementation Plan

> Branch: `fitur/ui-mobile`. Task berurutan; tiap task: implement → cek di browser
> (DevTools mode HP 360×800) → test bila ada logika → commit.
> Basis desain: draft spec di study ketua (`2026-07-09-stoklens-mobile-ui.md`) — flat
> touch-first, disetujui 2026-07-09. Desain Stitch tim = bahan tweak visual belakangan,
> struktur & integrasi API dari plan ini.

**Goal:** Web app mobile-first (dipakai di Chrome Android) untuk seluruh alur:
KPI beranda, katalog + enrollment kamera, opname (foto/manual/video), laporan + export.
Semua endpoint sudah ada dan teruji — ini murni pekerjaan frontend + wiring.

**Stack:** Jinja2 templates + vanilla JS (fetch) + CSS custom tokens. TANPA framework
build (nol toolchain — sengaja, lihat keputusan arsitektur di spec).

## Design tokens (WAJIB dipakai konsisten — sinkron dengan prompt Stitch tim)

```css
:root {
  --primary:#2563EB; --cta:#F97316; --bg:#F8FAFC; --surface:#FFFFFF;
  --text:#1E293B; --text-2:#64748B; --border:#E2E8F0;
  --pos:#16A34A; --neg:#DC2626;           /* laba/hijau, rugi-shrinkage/merah */
  --r:12px; --sp:8px;                      /* radius, spacing unit 8dp */
}
@media (prefers-color-scheme: dark) { :root {
  --bg:#0F172A; --surface:#1E293B; --text:#F1F5F9; --text-2:#94A3B8; --border:#334155;
}}
```

Aturan non-negotiable: touch target ≥48px • angka rupiah `font-variant-numeric:tabular-nums`
• body 16px • ikon Lucide SVG inline (TANPA emoji) • warna semantik selalu + ikon/teks
• transisi 150–300ms + `prefers-reduced-motion` • tanpa horizontal scroll di 360px.

## File structure

```
stoklens/webui/
  __init__.py            # router FastAPI utk halaman (Jinja2Templates)
  templates/
    base.html            # skeleton + bottom-nav + FAB slot + toast container
    beranda.html  barang.html  barang_detail.html
    opname.html   laporan.html  laporan_detail.html
  static/
    tokens.css  app.css  app.js  icons.svg (sprite Lucide)
tests/test_webui.py      # halaman ke-render + berisi elemen kunci (TestClient)
```

`api.py`: mount `StaticFiles` + include router webui; dashboard HTML lama dihapus
(diganti beranda). Tambah `jinja2` ke requirements + CI deps.

---

### Task 1: Fondasi — base.html + tokens + bottom nav + router

- Router `webui/__init__.py`: `GET /ui/*` render template; redirect `/` → `/ui/beranda`.
- `base.html`: viewport meta, safe-area padding (`env(safe-area-inset-*)`), bottom nav
  4 item (Beranda/Barang/Opname/Laporan — ikon+label, item aktif ber-highlight),
  slot FAB, container toast (`aria-live="polite"`).
- `app.js`: helper `api(path, opts)` (fetch + error toast), `rp(n)` format rupiah.
- Test: `GET /ui/beranda` → 200, mengandung `nav`, 4 label menu.
- Commit: `feat(ui): fondasi layout mobile + design tokens + bottom nav`

### Task 2: Beranda (GET /api/dashboard)

- 3 kartu KPI: Nilai stok / Shrinkage terakhir / Potensi rugi expired — angka besar
  tabular, tap → drill ke laporan/barang terkait.
- Daftar peringatan: `stok_menipis[]` (badge merah + qty/minimum), ringkasan
  `scan_terakhir` (tanggal, tipe, selisih Rp).
- Empty state ramah kalau belum ada data ("Mulai dengan daftarkan barang →" + tombol).
- Commit: `feat(ui): beranda KPI + peringatan`

### Task 3: Barang — list (GET /api/products)

- List card: nama, qty besar, harga modal→jual, badge margin %; sort nama; search
  client-side (filter di JS, data ≤ ratusan).
- FAB "+ Barang" → Task 4. Tap card → Task 5.
- Commit: `feat(ui): daftar barang`

### Task 4: Form enrollment (POST /products multipart)

- Field: nama*, harga modal*, harga jual, stok awal, stok minimum. Margin % dihitung
  live saat ketik. Label terlihat (bukan placeholder-only), keyboard numerik
  (`inputmode="numeric"`), error di bawah field.
- Foto: `<input type="file" accept="image/*" capture="environment" multiple>` —
  preview thumbnail + hapus/retake, minimal 1 foto, saran UI "3–5 sudut".
- Submit: tombol disabled+spinner saat upload (CLIP butuh beberapa detik) → toast
  sukses → kembali ke list.
- Commit: `feat(ui): enrollment barang via kamera`

### Task 5: Detail barang + kartu stok + penyesuaian

- Header: nama, qty, nilai Rp, margin; edit inline harga/minimum (PATCH).
- Ledger (`GET /api/products/{id}` → `ledger[]`): riwayat qty + sumber + alasan + tanggal.
- Sheet "Penyesuaian stok": stepper besar ±, pilihan alasan (rusak/hilang/retur/koreksi)
  → `POST /api/adjustments`; error 400 (stok negatif) tampil jelas.
- Commit: `feat(ui): detail barang + kartu stok + penyesuaian`

### Task 6: Opname — hub + manual (POST /api/opname-manual)

- Halaman Opname: 3 kartu mode — **Foto (utama/default)**, Manual, Video — dengan
  penjelasan 1 kalimat kapan dipakai.
- Manual: checklist semua barang, stepper qty fisik 48px, progress "12/30 diisi",
  submit → laporan selisih → dialog "Terapkan ke buku stok?" (`terapkan`).
- Commit: `feat(ui): opname manual`

### Task 7: Opname foto & video (POST /api/scans-foto, POST /scans)

- Foto: `capture="environment"` multi-shot, grid preview per sub-segmen + hint SOP
  ("1 foto = 1 sub-segmen, jangan overlap"), opsi guided (pilih produk per blok),
  upload progress → hasil + breakdown per foto (`per_foto`) buat spot dobel.
- Video: `accept="video/*" capture` / pilih file, pilih `count_mode` (line default +
  penjelasan), progress panjang (proses bisa >1 menit — pesan "sedang dihitung, jangan
  tutup") → hasil.
- Commit: `feat(ui): opname via foto dan video`

### Task 8: Laporan + export

- List scan (perlu endpoint kecil baru: `GET /api/scans` — id, tanggal, tipe, lokasi,
  total selisih Rp; tulis + test dulu ala TDD seperti endpoint lain).
- Detail: tabel per barang (tercatat/fisik/selisih/shrinkage/expired) — angka tabular,
  baris merah/hijau + ikon; link export `GET /api/export/stok.csv`.
- Commit: `feat(ui): laporan opname + export`

### Task 9: QA mobile + polish (checklist sebelum PR)

- [ ] 360×800 & 412×915 & landscape: tanpa horizontal scroll, form tidak ketutup keyboard
- [ ] Dark mode: kontras teks ≥4.5:1 (cek DevTools), kedua tema diperiksa
- [ ] Semua touch target ≥48px; press feedback ≤150ms
- [ ] Skeleton/spinner di semua fetch; empty state semua halaman; toast auto-dismiss 4s
- [ ] Konfirmasi sebelum aksi destruktif (terapkan opname, penyesuaian besar)
- [ ] `pytest` hijau (test_webui + suite lama)
- Commit: `polish(ui): QA mobile pass`
